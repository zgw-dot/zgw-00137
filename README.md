# 🏭 仓库拣货调度大师

一款本地可玩的仓库拣货调度策略小游戏。玩家需要合理安排拣货员的路线，分配推车资源，在截止时间内完成所有订单。

## 🎮 游戏特性

### 核心玩法
- **拣货员调度**：安排拣货员前往货架取货，然后送到打包台
- **推车管理**：合理分配有限的推车资源（使用推车可获得1.5倍分数加成）
- **路线规划**：避开单向巷道冲突，不能让两名拣货员在单向巷道中相向而行
- **连单奖励**：同一拣货员连续完成订单可获得额外加成（最高2倍）
- **时间压力**：每个订单都有截止时间，超时会扣分

### 游戏系统
- ✅ **两个内置关卡**：从简单到困难，循序渐进
- 💾 **进度保存**：自动保存关卡进度、最高分和设置
- 🎬 **失败回放**：失败后可以回放关键步骤，分析错误原因
- 📤 **关卡导入导出**：支持自定义关卡的JSON导入导出
- ⚙️ **游戏设置**：可调节游戏速度、网格显示、音效开关等
- 📱 **响应式设计**：支持不同屏幕尺寸

### 失败保护机制
- 🚫 **单向巷道冲突检测**：禁止两名拣货员同时进入同一单向巷道相向而行
- 🚫 **推车不足检测**：没有可用推车时不允许分配
- 🚫 **超时订单拦截**：订单超时后无法继续出库
- 🚫 **未知库区检测**：导入关卡时验证所有货架是否存在

## 🎯 游戏操作

### 主界面
- **当前订单**：显示所有订单及其剩余时间、分数
- **占用路线**：显示正在执行任务的拣货员路线
- **资源状态**：实时显示可用推车和拣货员数量
- **倒计时**：显示关卡剩余时间

### 操作步骤
1. 选择一个空闲的拣货员
2. 选择一个待处理的订单
3. 选择是否使用推车（勾选框）
4. 点击"🚀 派遣拣货"按钮
5. 观察拣货员移动、拣货、打包的全过程
6. 完成所有订单即可通关

### 界面元素说明
- 🏁 **蓝色方块**：拣货员出生点
- 📦 **绿色方块**：打包台（完成订单的终点）
- 📦 **灰色方块**：货架（订单取货点）
- ⬜ **浅色方块**：双向巷道（可自由通行）
- 🟧 **橙色渐变**：单向巷道（箭头指示通行方向）
- 🔵 **蓝色圆形**：空闲拣货员
- 🟠 **橙色圆形**：忙碌拣货员
- 🟢 **绿色方块**：可用推车
- 💠 **蓝色光点**：拣货员预定路线

## 📊 评分规则

| 项目 | 说明 |
|------|------|
| 基础分 | 每个订单基础100分 + 每件商品20分 |
| 时间奖励 | 剩余秒数 × 0.5分 |

| 完美奖励 | 剩余时间超过50%且完成所有订单 +500分 |

## 🎯 游戏技巧

1. **优先处理紧急订单**：注意红色标记的紧急订单（剩余时间<30秒）
2. **利用连单奖励**：尽量让同一个拣货员连续完成订单
3. **合理使用推车**：推车有1.5倍加成，但数量有限，留给高分订单
4. **注意单向巷道**：橙色箭头指示方向，不要让两个拣货员对着走
5. **提前规划路线**：派遣前看一下占用路线，避免路线冲突
6. **调整游戏速度**：新手可以调慢速度，高手可以加速挑战

## 🔧 技术说明

- **纯前端实现**：无需后端服务器，HTML + CSS + 原生JavaScript
- **本地存储**：使用localStorage保存游戏进度

- **冲突检测**：实时检测单向巷道冲突
- **无依赖**：无需安装任何npm包，开箱即用
